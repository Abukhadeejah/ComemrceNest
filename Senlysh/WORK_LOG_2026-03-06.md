# Work Log — 2026-03-06

## Summary
Implemented cashback rule changes (GST-net + delayed credit), added automated cashback release scheduling, rebuilt order details/invoice flows, and fixed admin/tenant order-detail routing so order links resolve correctly.

## What Was Implemented

### 1) Cashback business rules
- Cashback now credits after a 5-day delay instead of immediate wallet credit.
- Cashback base now supports GST-excluded calculation (cash paid minus GST component).
- Cashback processing remains idempotent to avoid duplicate credits.

### 2) Cashback release automation
- Added secure cron endpoint to release due cashback in batches.
- Added Vercel cron schedule to run release job hourly.
- Added authorization guard using `CRON_SECRET` for production requests.

### 3) Customer/admin order data improvements
- Customer orders API now returns real DB-backed payment/cashback/item fields.
- Order details pages show ordered products from DB (name, SKU, variant, quantity, pricing).
- Invoice download flows standardized to `/api/orders/{orderId}/invoice`.

### 4) Dedicated Order Details pages (dynamic)
- Added global dynamic route: `/admin/order-details/[id]`.
- Added tenant dynamic route: `/{tenant}/admin/order-details/[id]`.
- Both routes load live order + order_items from DB and support invoice download.

### 5) Routing/link fixes
- Shared order table updated to support separate list path vs detail path.
- `View Details` links now route to dedicated order-details paths.
- Legacy detail routes under `/admin/orders/[id]` and `/{tenant}/admin/orders/[id]` were converted into redirect aliases.
- Order ID (order number cell) in orders table made clickable to open order details directly.
- Admin customer order history order IDs and View Details links updated to open dedicated order details.
